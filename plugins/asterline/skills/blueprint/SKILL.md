# Blueprint

Use this skill when a change crosses modules, has launch risk, or lacks clear
acceptance criteria.

1. Read all local instructions that govern the target files.
2. Map the affected files, manifests, tests, and runtime surfaces.
3. Identify unknowns that cannot be resolved from the repository.
4. Define the plan as ordered tasks with explicit evidence for each task.
5. Include final verification lanes: static validation, runtime smoke, diff
   review, and user-facing proof.
6. Do not start implementation until the plan is approved when approval was
   requested.
