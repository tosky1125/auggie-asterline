# GIT-BASH REMINDER COMPONENT

## OVERVIEW

One-shot Windows Git Bash guidance hook. This source-bearing reminder is distinct from the sibling dist-only Git Bash MCP bundle.

## SURFACES

- `src/asterline-hook.ts`: platform detection, once-per-session marker, PostCompact reset.
- `src/cli.ts`: Node hook CLI; malformed input and operational errors fail open.
- `dist/`: tracked tsc output consumed by the installed wrappers.
- `hooks/hooks.json`: standalone `Bash` matcher, not the installed Auggie matcher.
- `../../mcp/git_bash/dist/`: separate Windows-only MCP generated upstream; never generated by this package.

## LOCAL CONTRACTS

- Runtime is Node >=20; tests use Bun.
- Preserve exact one-shot behavior per sanitized session and reset on PostCompact.
- Keep Windows detection compatible with `platform`, `OS`, `ComSpec`, and `SystemRoot` signals.
- Source changes require a wrapper-level payload smoke after the inherited dist regeneration step.
- The reminder may inject context but must never block or crash a turn.

## AGGREGATE INTEGRATION

Parent wiring adapts the public feature name to `git-flow` and matcher to Auggie `launch-process`. Current source accepts only tool name `Bash`; a synthetic `launch-process` payload produces no reminder. Treat that mismatch as an integration defect and require an end-to-end Auggie payload test when touching the adapter.

The separate MCP bundle is shipped but intentionally not registered on Linux. Its environment names and registration/stamping paths are not shared with this component.

## VALIDATION

```bash
npm run build
bun test test/*.test.ts
node dist/cli.js help
```

Then run the inherited plugin packaging gate; matcher/wrapper changes also require the inherited root marketplace validator.

## ANTI-PATTERNS

- Do not infer MCP behavior from reminder tests.
- Do not copy the standalone `^Bash$` matcher into the aggregate manifest.
- Do not hand-edit the sibling MCP dist bundle as a component fix.
